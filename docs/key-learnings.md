1. CTAN downloads can fail silently - Always verify downloaded .sty files aren't HTML error pages
2. tlmgr is reliable - For missing LaTeX packages, sudo tlmgr install is the safest approach
3. React hooks must be consistent - Never define hooks after conditional returns
4. XeLaTeX templates need fonts bundled - fontspec errors usually mean missing font files
5. Error context matters - Showing lines around errors helps debug LaTeX failures
6. Hardcoded URLs block deployment - Always use environment variables (VITE_API_URL, CORS_ORIGINS)
7. Duplicate config drifts apart - Centralize configuration in a single source of truth (e.g., config/templates.py)
8. Dead code accumulates silently - Regularly audit for unused imports, endpoints, and UI fields
9. Error boundaries prevent full crashes - Add React ErrorBoundary early, not after users report crashes
10. CORS wildcards are lazy - Specify exact methods/headers needed (GET, POST vs "*")
11. Sanitize user input before LaTeX compilation - Block \write18, \openin, \catcode and similar shell-escape commands
12. Use Literal types for enums in Pydantic - Literal["user", "assistant"] catches invalid values at validation time
13. Large CSS files become unmaintainable - Split into component CSS (see TemplateSelector.css pattern) or use CSS Modules
14. Custom hooks reduce component complexity - Extract related state into useTemplates, useCompiler, useChat patterns
15. Hook return objects cause infinite loops - useApi() returns new object each render; remove from useEffect deps or use useMemo
16. Sequential string replacement breaks LaTeX escaping - Replacing `\` first then `{` re-escapes the `{` in `\textbackslash{}`. Use a single-pass regex (`re.compile` + `re.sub` with a dict lookup) to escape all special chars in one pass.
17. useRef mirrors state for stable event listeners - `addEventListener` callbacks capture the value at registration time (stale closure). Mirror state with a `useRef` (update both in setState calls) so the callback reads `ref.current` which is always fresh. Pattern: `const widthRef = useRef(initial); setWidth(n => { widthRef.current = n; return n; })`.
18. onDragEnter not onDragOver for drop-target state - `onDragOver` fires every frame (up to 60fps) while dragging; use `onDragEnter` (fires once per element boundary crossing) to update `dragOver` state. Always call `e.preventDefault()` in `onDragOver` to allow dropping.
19. Jinja2 custom delimiters avoid LaTeX brace conflicts - Default `{{ }}` / `{% %}` clash with LaTeX `{}`. Configure `Environment(variable_start_string='(( ', variable_end_string=' ))', block_start_string='(% ', block_end_string=' %)')` to use `(( var ))` / `(% block %)` syntax.
20. isDirty tracking via useEffect + ref - To know if data changed since last action (e.g., last compile), store `lastRef.current = data` on action completion and track changes with `useEffect(() => { if (lastRef.current !== null) setIsDirty(true); }, [data])`. Cleaner than threading a dirty flag through every mutation handler.
21. draggable={true} in JSX breaks cursor positioning in child inputs - The browser modifies mouse event handling for all descendants of a draggable element, not just for drag events. Even if you cancel `dragstart` with `e.preventDefault()`, subsequent clicks to reposition the text cursor in inputs inside the card fail. Fix: omit `draggable` from JSX entirely. In the grip handle's `onMouseDown`, imperatively set `el.draggable = true` via `closest('[data-drag-card]')`. Reset to `false` in `onDragEnd`. The element is only draggable during the brief drag gesture.
22. \enspace followed by a letter creates an undefined command - In LaTeX, `\enspace` is a control word. Control words end at the first non-letter, so `\enspaceLondon` is parsed as the command `\enspaceLondon` (undefined). Always terminate `\enspace` (and any control word that should not absorb the following letter) with `{}`: write `\enspace{}` before text content. `{` is not a letter so it ends the command name without producing output.
23. Jinja2 join + latex_escape ordering matters - When using `join` with a separator that contains LaTeX commands (e.g., `\textbullet{}`), you must `map('latex_escape')` individual items FIRST, then `join` with the raw LaTeX separator. Doing `join | latex_escape` escapes the separator itself. Safe separators like `, ` or `; ` can use `join | latex_escape` since they contain no LaTeX special chars.
24. LaTeX \href{} needs lighter escaping than body text - The hyperref package handles most URL characters in `\href{url}{text}`. Only `%` (starts a LaTeX comment) and `#` (parameter character) need escaping in the URL argument. Using the full `latex_escape` on URLs would break them (e.g., escaping `_` to `\_` corrupts URLs).
25. LaTeX \begin{list} requires at least one \item - An empty list environment (zero items) is invalid LaTeX and will fail compilation. Template conditionals must guard against rendering list environments when there are no items to render.
26. Compilation tests catch bugs rendering tests miss - The Jinja2 rendering tests verified template output looked correct, but the actual pdflatex/xelatex compilation tests caught a real bug (empty `rSubsection` list) that only manifests at compile time. Both test layers are necessary.
27. Deedy contact_parts array pattern for clean separators - Building an array of contact items then `join`-ing with the separator avoids leading/trailing separators when optional fields are missing. Cleaner than inline `if not loop.first` checks when items are conditionally included.
28. RAF ID in useRef prevents stale closures in drag handlers - When using requestAnimationFrame in event listeners registered inside a callback (like onMouseMove in startResize), declaring `let rafId` in the callback creates a new closure scope per drag. Subsequent onMouseMove calls can't access or cancel the previous RAF, causing updates to queue at panel limits. Fix: `const rafIdRef = useRef<number | null>(null)` at component level, then always cancel `rafIdRef.current` before scheduling new RAF.
29. Window mouseleave forces cleanup for drag-outside-viewport - Drag operations that reach window boundaries (e.g., resizing a panel to the edge) can cause the cursor to leave the browser viewport. If mouseup fires outside the window, cleanup listeners don't trigger, leaving the operation stuck. Add `window.addEventListener('mouseleave', onMouseLeave)` that calls the cleanup function if still in dragging state. Remove this listener in cleanup alongside mousemove/mouseup.
30. State isolation requires explicit clearing between flows - When an app has multiple entry paths (e.g., "Build my CV" vs "Import CV"), shared state like formData can leak between flows if not explicitly cleared. Always reset relevant state when entering a flow: set formData/importResult/selectedTemplate to null in the entry point handlers. Also clear on Back navigation to prevent stale data persisting when user changes their mind mid-flow.
31. String.includes() is too broad for URL protocol detection - Using `label.includes('://')` to detect URLs has false positives (e.g., "Contact at: example.com://support" matches). Use `label.startsWith('http://')` or `label.startsWith('https://')` for precise protocol detection. Also guard with `label.trim() === ''` for whitespace-only strings and check `link.url` exists before deriving labels.
32. Iframes capture mouse events during drag operations - When an iframe (e.g., PDF preview) is present in a resizable panel, dragging the resize handle fails when the cursor moves over the iframe. The iframe intercepts mousemove events, preventing them from reaching document listeners. Fix: add `pointer-events: none` to the iframe during drag via state flag (`isResizing`). Set the flag true in drag start, false in drag end, apply `style={{ pointerEvents: isResizing ? 'none' : 'auto' }}` to iframe. This makes the iframe transparent to events during drag while preserving interactivity when not dragging.
33. LaTeX \savebox can measure tabu table height for auto-detecting single vs multi-line headers - Compare against 1.5x and 2.5x \baselineskip thresholds to select correct vspace adjustment. Eliminates manual line count parameters in Jinja2 templates. Pattern: `\savebox\headerbox{\begin{tabu}...\end{tabu}}` then `\ifdim\ht\headerbox<1.5\baselineskip ... \else\ifdim\ht\headerbox<2.5\baselineskip ... \else ... \fi\fi`.
34. Pipecat PipecatClient requires ProtobufFrameSerializer for Nova Sonic - Sample rates: 16kHz input (user mic), 24kHz output (assistant). WebSocket URLs must use wss:// in production HTTPS contexts; use environment variables not hardcoded ports. Optional dependencies: wrap Pipecat imports in try/except so app starts without it.
35. React createPortal is the right pattern for rendering overlay content into a different DOM container - VoiceWidget overlay portal renders into navWrapper div outside the form builder component tree. Use `ReactDOM.createPortal(<OverlayContent />, document.getElementById('navWrapper'))` to break out of parent component's DOM hierarchy while maintaining React event bubbling and context.
36. WebSocket URLs must use wss:// in production HTTPS contexts - Browser blocks ws:// connections from HTTPS pages. Use environment variables (VITE_WS_URL) to switch between ws://localhost:8000 (dev) and wss://domain.com (prod). Never hardcode ports or protocols in WebSocket connection URLs.
