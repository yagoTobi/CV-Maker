# Phase 5 — UI Design Contract

> Visual and interaction contract for the AI Integration phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase introduces four new UI surfaces: **EditorToolbar**, **ChangePanel** (with ChangeCards), **ImportToast**, and a **read-only MedLengthTemplate** variant. It also restructures the Apply to Job step 3 from a narrow card layout to a wide two-panel view.

## Interaction Contracts

### Scroll Synchronization (D-02)

- **Mechanism:** `IntersectionObserver` watches `[data-section]` elements on the CV. When a section becomes the topmost visible section (threshold 0.3), the panel scrolls to the first `[data-change-section]` card matching that section key.
- **Direction:** CV scroll drives panel scroll (one-way). Panel scroll does NOT drive CV scroll.
- **Anti-jitter:** `isAutoScrolling` ref flag. Set to `true` before programmatic `scrollIntoView`, cleared 150ms after scroll completes. Observer callbacks are skipped while flag is `true`.
- **Scroll behavior:** `scrollIntoView({ behavior: 'smooth', block: 'nearest' })` for panel auto-scroll.

### Accept/Reject Flow (D-03, D-04)

1. User clicks "Accept" on a ChangeCard.
2. `useTailor.acceptChange(changeId)` is called (queued to prevent race conditions).
3. Card visually transitions: buttons replaced by "Undo" link, left border turns green, card content fades slightly.
4. The CV (if not read-only) re-renders with the accepted change applied via `applyTailorChanges()`.
5. User clicks "Reject" on a ChangeCard.
6. `useTailor.skipChange(changeId)` is called.
7. Card fades to 0.5 opacity, left border turns muted gray, buttons replaced by "Undo" link.
8. User clicks "Undo" on an accepted or rejected card.
9. `useTailor.undoChange(changeId)` is called. Card restores to pending state with Accept/Reject buttons.

### Editable Suggestion (D-03)

1. User clicks "Edit suggestion" link on a pending ChangeCard.
2. A `<textarea>` appears below the diff view, pre-filled with the current suggestion text.
3. User edits the text. On blur, `useTailor.editChangeValue(changeId, newValue)` is called.
4. The diff view updates to show the new suggestion text vs. original.
5. User can then Accept (applies edited version) or Reject as normal.

### Import Flow (D-06, D-07)

1. User clicks "Import CV" in the EditorToolbar.
2. A file picker opens (accept: `.pdf,.docx,.json`).
3. On file selection, `useImport.importFile()` is called. Toolbar shows "Extracting CV data..." spinner next to import button.
4. On success: extracted formData is set via `useDirectEditor` (or context). ImportToast appears at top.
5. The CV re-renders with imported data. User can begin editing immediately.
6. ImportToast auto-dismisses after 8 seconds or on user click of dismiss button.
7. On failure: ImportToast appears with error message in `--error` color. Does not auto-dismiss on error (user must dismiss manually).

### Panel Open/Close

1. When tailor suggestions arrive (or user triggers tailoring), the ChangePanel slides in from the right over 200ms.
2. The CV container's right margin adjusts to make room (preventing overlap). On viewports >= 1200px, the CV remains centered in the remaining space. On viewports < 1200px, the panel is a bottom sheet.
3. Toggle button (chevron circle at panel edge) allows manual close. Clicking it slides the panel out. Clicking again slides it back in.
4. Panel close does NOT discard pending suggestions. State is preserved in `useTailor`.

## Per-Field Confidence Hints (D-08 -- Claude's Discretion)

**Decision:** Do NOT render per-field confidence hints in this phase. Rationale:
1. The ImportToast already communicates overall confidence level (high/medium/low).
2. Per-field underlines on the CV document would conflict with the clean editing surface (D-01 principle: "the CV stays clean and readable").
3. Low-confidence fields are rare in practice (most imports are high confidence).
4. Adding underlines would require MedLengthTemplate to be aware of import metadata, violating the principle that MedLengthTemplate renders formData only.

If user feedback later indicates this is needed, it can be added as a v2 enhancement by passing a `confidenceMap` prop to MedLengthTemplate that renders subtle dotted underlines on low-confidence fields.
