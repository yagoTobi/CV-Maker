## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps (e.g., NavBar logo icon padding) |
| sm | 8px | NavBar internal element gaps, button icon-to-text gap |
| md | 16px | NavBar horizontal padding, default element spacing |
| lg | 24px | Template card info padding, section gaps |
| xl | 32px | Page content side margins |
| 2xl | 48px | NavBar height (matches current EditorToolbar) |
| 3xl | 64px | Not used in this phase |

Exceptions: NavBar height is exactly 48px to match the current EditorToolbar height. This preserves `usePageBreak` calculations (Pitfall #4 from RESEARCH.md). No deviation allowed.

## Component Inventory

### New Components

#### 1. NavBar

**File:** `frontend/src/components/NavBar.tsx` + `NavBar.module.css`

**Placement:** Rendered inside `WorkingLayout` component above `<Outlet />`. NOT rendered on landing page (`/`).

**Layout:** Single horizontal bar, 48px height, flexbox with space-between.

| Zone | Content | Alignment |
|------|---------|-----------|
| Left | "CV Maker" logo text (clickable, navigates to `/`) + "My CVs" link (navigates to `/dashboard`) | `flex` with `gap: 16px`, vertically centered |
| Right (editor page) | "Import CV" ghost button + "Download PDF" accent button + SaveIndicator | `flex` with `gap: 8px`, vertically centered |
| Right (other pages) | "+ New CV" ghost button | `flex` with `gap: 8px`, vertically centered |

**Visual spec:**

```
|  CV Maker   My CVs                    Import CV  [ Download PDF ]  Saved  |
|  ^^logo     ^^link                    ^^ghost     ^^accent-fill    ^^save |
```

- Background: `var(--bg-secondary)` (#FFFFFF)
- Border bottom: `1px solid var(--border-color)` (#E2E8F0)
- Padding: `0 16px`
- Position: `sticky`, `top: 0`, `z-index: 50`
- "CV Maker" text: 16px, weight 600, color `var(--text-primary)`, cursor pointer
- "My CVs" link: 14px, weight 400, color `var(--text-secondary)`, hover color `var(--accent)`
- Ghost buttons (Import CV, + New CV): transparent background, `1px solid var(--border-color)`, `border-radius: var(--radius-sm)` (6px), padding `6px 16px`, font 13px weight 500, color `var(--text-secondary)`, hover background `var(--bg-hover)`, hover color `var(--accent)`
- Accent button (Download PDF): background `var(--accent)` (#3B82F6), color `#FFFFFF`, no border, `border-radius: var(--radius-sm)` (6px), padding `6px 16px`, font 13px weight 500, hover background `var(--accent-hover)` (#2563EB)
- SaveIndicator: existing component, rendered inline (already has `inline` prop)
- Disabled state (while importing/downloading): opacity 0.5, cursor not-allowed
- Loading state on buttons: 14px spinner + loading text (same pattern as EditorToolbar)
- Transition: `background 150ms ease, color 150ms ease` on all interactive elements

**Route detection:** Use `useLocation().pathname === '/build/form'` to determine which right-side action set to render.

**Editor actions pattern:** DirectEditPage provides `handleImportClick`, `handleDownload`, `saveStatus`, `isImporting`, `isDownloading` via a lightweight `EditorActionsContext`. NavBar reads this context. When context has no actions (non-editor pages), NavBar renders "+ New CV" button.

Source: D-08, D-09, D-10, D-11, D-12, D-16, D-17, RESEARCH.md Pattern 2/3.

#### 2. WorkingLayout

**File:** `frontend/src/components/WorkingLayout.tsx` + `WorkingLayout.module.css`

**Purpose:** React Router layout route component. Wraps all routes except landing with NavBar.

**Structure:**
```tsx
<>
  <NavBar />
  <Outlet />
</>
```

No additional styling beyond NavBar and Outlet composition. The Outlet renders the child route component directly.

Source: RESEARCH.md Pattern 1 (layout route with Outlet).

#### 3. EditorActionsContext

**File:** `frontend/src/contexts/EditorActionsContext.tsx`

**Purpose:** Lightweight context for DirectEditPage to pass its toolbar actions (import, download, save status) up to NavBar without prop drilling through Outlet.

**Shape:**
```typescript
interface EditorActions {
  onImport: () => void;
  onDownload: () => void;
  saveStatus: SaveStatus;
  isImporting: boolean;
  isDownloading: boolean;
}
```

**Provider:** Wraps Outlet inside WorkingLayout. Default value: `null` (no editor actions). NavBar checks `if (editorActions)` to decide which right-side buttons to render.

**Consumer:** NavBar reads via `useEditorActions()` hook.

**Setter:** DirectEditPage calls `useSetEditorActions()` on mount to register its handlers, and cleans up on unmount.

### Modified Components

#### 4. TemplateSelector -- "Coming soon" badge + disabled state

**Additions to existing TemplateSelector.tsx and TemplateSelector.module.css:**

**Disabled card:**
- CSS class `.disabled`: `opacity: 0.55`, `cursor: default`, `pointer-events: none`
- No hover transform, no hover border change, no overlay with "Use This Template" button
- Filter: `grayscale(0.2)` on the preview image

**"Coming soon" badge:**
- Position: absolute, top-right corner of `.previewWrapper`, `top: 12px`, `right: 12px`
- Background: `var(--accent-light)` (#DBEAFE)
- Border: `1px solid var(--accent)` (#3B82F6)
- Border radius: `var(--radius-sm)` (6px)
- Padding: `4px 8px`
- Text: "Coming soon", 11px (0.6875rem), weight 600, uppercase, letter-spacing 0.1em, color `var(--accent)`
- Z-index: 2 (above preview image)

**Guard logic:** `SUPPORTED_TEMPLATES = new Set(['med-length-proff-cv'])`. Non-supported templates get `.disabled` class and badge. `handleSelect` returns early for unsupported templates.

Source: D-05, D-06, RESEARCH.md template selector code example, existing eyebrow badge pattern in TemplateSelector.

#### 5. DirectEditPage -- EditorToolbar removal

**Changes:**
- Remove `<EditorToolbar>` JSX from DirectEditPage render output
- Move `handleImportClick`, `handleDownload`, `saveStatus`, `isImporting`, `isDownloading` into `EditorActionsContext` via `useSetEditorActions()`
- Keep `ImportToast`, `MedLengthTemplate`, and all other content unchanged
- The `.page` CSS class remains; `padding-top` adjustment unnecessary since NavBar is sticky above and EditorToolbar was also sticky above (same 48px height)

Source: D-09, current EditorToolbar.tsx analysis.

#### 6. Dashboard -- handleApplyToJob navigation fix

**Changes:**
- `handleApplyToJob` navigates to `/apply` with `{ state: { baseVersionId: baseId } }` instead of `/build/form` with `{ state: { mode: 'tune' } }`
- Remove `handleVersionLoad`, `setSelectedTemplateForBuild` calls from handleApplyToJob (not needed -- ApplyToJobScreen fetches its own version data)

Source: D-14, RESEARCH.md Dashboard handleApplyToJob Fix.

#### 7. App.tsx -- Route restructure

**Changes:**
- Landing page (`/`) sits outside WorkingLayout (no nav bar)
- All other routes wrapped in `<Route element={<WorkingLayout />}>`
- Remove `/direct-edit` route and lazy import
- Remove `/import` route and lazy import
- Replace CVFormBuilder lazy import with DirectEditPage at `/build/form`
- Add WorkingLayout lazy import
- Keep FeatureErrorBoundary wrapping on all child routes
- NotFound route (`*`) sits outside WorkingLayout

Source: D-01, D-02, D-03, D-04, RESEARCH.md Route Swap in App.tsx.

## Accessibility

| Element | ARIA / Keyboard Contract |
|---------|--------------------------|
| NavBar | `<nav>` element with `aria-label="Main navigation"` |
| NavBar logo | `<a>` or `<button>` with accessible name "CV Maker - Go to home" |
| NavBar "My CVs" | `<a>` or `<button>` with `aria-current="page"` when on dashboard |
| NavBar buttons | Standard `<button>` elements with `aria-disabled` when loading |
| "Coming soon" badge | Decorative (parent card has `aria-disabled="true"`) |
| Disabled template card | `aria-disabled="true"`, `tabindex="-1"` to remove from tab order |
| WorkingLayout | No ARIA role needed (structural wrapper only) |

Keyboard navigation: All NavBar interactive elements must be focusable via Tab and activatable via Enter/Space. Disabled template cards are skipped in tab order.
