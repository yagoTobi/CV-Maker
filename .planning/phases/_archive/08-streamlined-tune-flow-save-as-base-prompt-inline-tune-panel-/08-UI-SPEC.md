## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Pill gaps, inline icon padding, summary row icon-text gap |
| sm | 8px | Compact element spacing, tier header internal padding, input gaps |
| md | 16px | Default element spacing, panel internal padding, tier body padding |
| lg | 24px | Section gaps between tiers, gap between CV and tune panel |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Loading/empty state vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions: none. Existing codebase uses rem-based spacing (0.75rem = 12px, 1.25rem = 20px) -- new components in this phase MUST use the same rem convention. 12px and 20px are permitted as existing codebase exceptions.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 13px (0.8125rem) | 400 | 1.3 |
| Body / Heading | 15px (0.9375rem) | 400 (body) / 600 (heading) | 1.5 (body) / 1.3 (heading) |
| Display | 24px (1.5rem) | 600 | 1.2 |

3 declared sizes: 13px, 15px, 24px. 2 declared weights: 400 (regular), 600 (semibold).

Body and Heading share 15px and are differentiated by weight alone. This matches the existing ChangePanel pattern (15px at both regular and semibold). The 2px gap between Label (13px) and Body (15px) provides observable hierarchy at small scale; the 9px gap between Body (15px) and Display (24px) creates strong visual separation for the score circle.

Role assignments:
- **13px / 400:** Labels, collapsed tier summaries, breadcrumb text, step number circles, pill text, muted secondary info
- **15px / 400:** Form input text, textarea content, paragraph descriptions, error/empty state body copy
- **15px / 600:** Tier titles, section group headings, button labels, breadcrumb active segment
- **24px / 600:** Score circle number (display-only, single use)

Source: Existing ChangePanel uses 13px/15px/24px. ApplyToJobScreen uses 0.8125rem (13px) for labels, 0.9375rem (15px) for step titles. Display 24px from ChangePanel score circle.

All text in the tune panel uses IBM Plex Sans (inherited from body). CV content uses EB Garamond (scoped to MedLengthTemplate).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF (--bg-secondary) | Tune panel background, tier card backgrounds, input backgrounds |
| Secondary (30%) | #F8FAFC (--bg-primary) | Page background behind panel, input field fills, collapsed tier hover |
| Accent (10%) | #3B82F6 (--accent) | "Analyze Match" button, "Save Tailored CV" button, active tier step number ring, "Tuning" badge in NavBar, focused input border |
| Destructive | #EF4444 (--error) | Not actively used in this phase (no destructive actions in the tune panel itself) |

Accent reserved for:
- Primary action buttons ("Save as Base CV", "Analyze Match", "Save Tailored CV")
- Active tier indicator ring (step number circle with accent border)
- "Tuning" badge dot/text in NavBar
- Focused input border-color
- Score circle border (via --success / --warning / --error, not accent)

Additional semantic colors used:
- --success (#10B981): Completed tier checkmark circle fill, score >= 80%, score delta positive
- --warning (#F59E0B): Score circle border 60-79%
- --error (#EF4444): Score circle border < 60%, missing requirement pills
- --success-light (#D1FAE5): Matched pill background
- --error-light (#FEE2E2): Missing pill background

Source: variables.css, existing ApplyToJobScreen + ChangePanel patterns.

### CV Area When Panel is Open

The CV container does NOT resize. The panel overlays from the right. The existing `contentArea` flex layout remains unchanged. The CV content naturally centers itself in the available viewport width. This matches the existing ApplyToJobScreen Step 3 pattern where `padding-right: 392px` was used on `.twoPanelContent` -- in this phase, we apply `padding-right: 416px` (400px panel + 16px breathing room) to `.contentArea` when the tune panel is open, via a CSS class toggle.

At viewport widths below 1200px, the panel becomes a bottom sheet (50vh height, full width) matching the existing ChangePanel responsive pattern.

## Interaction Contract

### Tier 1: Save as Base CV

**Trigger:** Tune panel opens AND `activeVersion` is null (unsaved CV).

**Layout:**
- Step number "1" with active styling
- Title: "Save as Base CV"
- Body: Single name input + primary button

**Input:**
- Name input: `padding: 8px 12px`, `font-size: 15px`, `border: 1px solid var(--border-color)`, `border-radius: var(--radius-sm)`, placeholder: "e.g., Creative CV"
- Auto-focused on panel open

**Action button:**
- Label: "Save as Base CV"
- Style: `.primaryBtn` from ApplyToJobScreen (full-width, accent background, white text, 15px weight 600)
- Disabled when: name input is empty
- Loading state: spinner + "Saving..."

**After save:**
- Tier 1 collapses to summary row: checkmark circle + "Save as Base CV" title + "Base CV: [name]" summary on right
- Tier 2 automatically expands
- Transition: 200ms ease, tier body slides up, next tier body slides down

**Skip condition:** If `activeVersion` is non-null when panel opens, Tier 1 renders only as a pre-completed summary row showing "Base CV: [activeVersion.name]". Tier 2 starts expanded.

### Tier 2: Job Details

**Trigger:** Tier 1 is completed (or was skipped because activeVersion exists).

**Layout:**
- Step number "2" with active styling (or "1" if Tier 1 was skipped -- but visually always shows the correct sequential number)
- Title: "Job Details"
- Body: Company + Role inputs (side-by-side grid), Job Description textarea, primary button

**Inputs:**
- Company Name: `width: 100%`, placeholder: "e.g., Google"
- Role: `width: 100%`, placeholder: "e.g., Senior SWE"
- Job Description: `min-height: 140px`, `resize: vertical`, placeholder: "Paste the full job description here..."
- All inputs use the same `.input` / `.textarea` styling from ApplyToJobScreen
- Company + Role in a 2-column grid: `grid-template-columns: 1fr 1fr`, `gap: 12px`

**Action button:**
- Label: "Analyze Match"
- Disabled when: job description is empty
- Loading state: spinner + "Analyzing..."

**After analysis completes:**
- Tier 2 collapses automatically to summary row: checkmark + "Job Details" + "[Company] [Role] [score]%" summary
- Tier 3 expands automatically with match analysis + change cards
- No explicit "Next" button needed -- transition is automatic when analysis and tailor suggestion fetch both complete

### Tier 3: Review Changes

**Trigger:** Analysis complete, tailor suggestions loaded.

**Layout:**
- Step number "3" with active styling
- Title: "Review Changes"
- Body: Match analysis summary (score circle + pills) + ChangeCard list + "Save Tailored CV" button

**Content:**
- Match summary: Reuse the existing ChangePanel match summary (score circle + matched/missing pills). Render directly in the tier body, NOT inside a nested ChangePanel -- but the ChangeCards are rendered using the existing ChangeCard component.
- Change cards: Grouped by section using `fieldPathToSection()`. Each group has an uppercase label. Cards have accept/reject/undo controls.
- "Accept All Remaining" button: full-width accent button, below all cards.
- "Save Tailored CV" button: fixed at bottom of the panel (sticky within the panel scroll area). Style: `.primaryBtn`, disabled when no changes accepted.

**Save action:**
- Loading state: spinner + "Saving..."
- After save completes: navigate to `/dashboard` with `location.state = { baseId: parentVersionId }`
- Panel closes automatically on navigation

### Panel Close

**Trigger:** X button at top-right of panel, or Escape key when panel has focus.

**X button:**
- Position: absolute, top: 16px, right: 16px within panel
- Size: 32px x 32px
- Style: no background, no border, `color: var(--text-secondary)`, hover: `color: var(--text-primary)`
- Icon: 16x16 X SVG (line x1=18 y1=6 x2=6 y2=18 + line x1=6 y1=6 x2=18 y2=18)

**Behavior:**
- Panel slides out via translateX animation
- CV area padding-right returns to 0
- NavBar "Tuning" indicator disappears
- Tier state is preserved (if user reopens, they continue from where they left off)

### NavBar Tuning Indicator

**When tune panel is open on editor page:**
- The "Tune for Job" ghost button changes to an active state: background: var(--accent-light), color: var(--accent), border-color: var(--accent)
- A small dot or text label is not needed -- the button highlight alone signals active state

**When tune panel is closed:**
- "Tune for Job" button returns to its normal ghost styling

### NavBar Non-Editor Pages (D-13)

- Remove the "+ New CV" button from the NavBar right group on non-editor pages
- Non-editor pages: NavBar right group is empty (just the left group with logo + "My CVs")
- This affects Dashboard, /apply redirect, and any other non-/build/form pages

### Dashboard Scoped View (D-11)

**Trigger:** Navigation to `/dashboard` with `location.state = { baseId: string }`.

**Behavior:**
- Dashboard reads `baseId` from location.state
- Filters `baseCvs` to show only the one matching base CV group (and its children)
- Shows a "breadcrumb" at top: "All CVs" link (clickable, clears filter) + separator + base CV name
- "All CVs" click: `navigate('/dashboard')` (no state) to show full list

**Breadcrumb styling:**
- Position: between header and CV groups
- "All CVs": font-size 13px, color: var(--accent), cursor: pointer, no underline, hover: underline
- Separator: " / " in var(--text-muted)
- Base CV name: font-size 13px, color: var(--text-primary), font-weight: 600

**When no state.baseId:** Dashboard renders normally (full list, no breadcrumb).

## State & Data Flow

### Tune Panel State (owned by DirectEditPage)

```
tunePanelOpen: boolean           -- panel visibility
tuneStep: 1 | 2 | 3             -- which tier is expanded
companyName: string              -- Tier 2 input
roleName: string                 -- Tier 2 input
jobDescription: string           -- Tier 2 input
matchAnalysis: MatchAnalysis | null  -- Tier 2 result
baselineScore: number            -- from match analysis
analyzing: boolean               -- Tier 2 loading
```

### useTailor Hook (reused as-is)

Initialized with `originalFormData` from the active version. `onApply` callback updates `previewFormData` state. In this phase, when the user is in Tier 3, the CV on the left switches to a read-only preview showing `previewFormData` (with accepted changes applied). This matches the existing ApplyToJobScreen Step 3 behavior.

### EditorActionsContext Extension

Add `isTuning: boolean` to the EditorActions interface. NavBar reads this to show active button state.
