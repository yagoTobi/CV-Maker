Gap 1 (BLOCKER - skills crash):
  File: MedLengthTemplate.tsx line 76
  Bug: `onFieldChange(\`skills[${skillIndex}].skills\`, JSON.stringify(updatedSkills))`

    **Problem:** EntryWrapper renders `<div class="entryWrap">` which inserts an extra div between the CSS grid parent (`.skillsGrid` / `.awardsGrid`) and the grid cells (SkillCategoryRow fragment children / AwardRow fragment children). CSS grid expects direct children to be grid cells, but EntryWrapper wraps each row's cells in a single div, collapsing the 2-column layout.

    **Solution: CSS subgrid on EntryWrapper for grid contexts.**

    Use `display: grid; grid-template-columns: subgrid; grid-column: 1 / -1` so EntryWrapper participates in the parent grid as a row spanning all columns, and its children become sub-grid cells. This preserves `position: relative` and `:hover` (unlike `display: contents` which removes the box model entirely). CSS subgrid is supported in Chrome 117+, Firefox 71+, Safari 16+ -- acceptable per the modern-browsers-only constraint in CLAUDE.md.

    Rejected alternatives: `display: contents` breaks :hover and position: relative for the delete button. Restructuring the grid per-row adds unnecessary complexity.

    **Implementation:**

    1. Update `EntryWrapper.tsx`:
       - Add `gridItem?: boolean` to `EntryWrapperProps`
       - Use `className={gridItem ? styles.entryWrapGrid : styles.entryWrap}` on root div
       - Delete button hover CSS works unchanged since the box model is preserved

    2. Update `EntryWrapper.module.css`:
       ```css
       .entryWrapGrid {
         display: grid;
         grid-template-columns: subgrid;
         grid-column: 1 / -1;
         position: relative;
       }

       .entryWrapGrid:hover > .deleteButton {
         opacity: 1;
       }
       ```

       - In `renderAwardsSection` (line ~539): add `gridItem={true}` to the EntryWrapper wrapping each award
