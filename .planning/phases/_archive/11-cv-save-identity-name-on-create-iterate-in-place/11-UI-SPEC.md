## Color

All color values reference existing design token variables. No new hex values introduced.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--bg-primary` | #F8FAFC | Page background |
| Secondary (30%) | `--bg-secondary` | #FFFFFF | NavBar surface, prompt dialog surface, dropdown surface |
| Accent (10%) | `--accent` | #3B82F6 | CV name button on hover/focus, OK button background, breadcrumb separator color |

Accent reserved for:
1. CV name button hover text color (matches existing `navLink:hover` pattern)
2. "OK" / "Save" button in name prompt (primary action)
3. Breadcrumb separator "/" character tint (subtle brand touch, see Breadcrumb section)

Secondary semantic note: `--text-secondary` (#64748B) for all non-primary label text ‚Äî breadcrumb company/role segment, dropdown items at rest, prompt subtext.

Dropdown overlay: no backdrop. The dropdown closes on outside click (click-outside handler) ‚Äî consistent with the project's preference for minimal overlays (ConfirmDialog has only 0.08 opacity backdrop when it needs one; name prompt uses a semi-transparent backdrop).

## States and Interactions

### NavBar Left Group ‚Äî State Machine

| Page / Condition | Left Group Content |
|------------------|--------------------|
| Non-editor page (Landing, Dashboard, TemplateSelector) | Logo only |
| Editor page, not tuning, no activeVersion | Logo + "Untitled CV ‚ñæ" (CVNameButton with placeholder name) |
| Editor page, not tuning, activeVersion exists | Logo + "[CV name] ‚ñæ" (CVNameButton) |
| Editor page, isTuning | Logo + breadcrumb "[base name] / [company or role]" |

### Name Prompt ‚Äî State Machine

| Trigger | State |
|---------|-------|
| First auto-save debounce fires, no activeVersion.id | Dialog opens, input auto-focused, text auto-selected |
| User types name and presses Enter or clicks "Save CV" | Dialog closes, name used for version creation, save proceeds |
| User clears input and presses Enter or clicks "Save CV" | Name defaults to "My CV", dialog closes, save proceeds |
| User clicks "Skip for now" or Escape or backdrop | Dialog closes, name defaults to "Untitled CV", save proceeds |
| Save completes | `activeVersion.id` is set; subsequent saves use PATCH (in-place update per D-02) |

### CV Switcher Dropdown ‚Äî Interaction Flow

| Action | Result |
|--------|--------|
| Click CVNameButton | Dropdown opens; current CV item is highlighted |
| Click another CV in list | Dropdown closes; `handleVersionLoad(version)` called; editor loads that CV |
| Click "+ New CV" | Dropdown closes; `resetForNewBuild()` + navigate to `/build` |
| Click outside dropdown | Dropdown closes |
| Escape key | Dropdown closes, focus returns to CVNameButton |
| Click CVNameButton again (open) | Dropdown closes |

## Integration Contracts

### EditorActionsContext ‚Äî Required Extensions

The following fields must be added to the `EditorActions` interface to enable NavBar breadcrumb rendering:

| Field | Type | Source |
|-------|------|--------|
| `cvName` | `string` | `activeVersion?.name ?? 'Untitled CV'` |
| `tuneCompanyName` | `string` | TunePanel tier 2 `companyName` state |
| `tuneRole` | `string` | TunePanel tier 2 `roleName` state |

NavBar reads these fields to render CVNameButton text and the breadcrumb sub-segment.

### useAutoSave ‚Äî Name Prompt Hook Point

`useAutoSave` receives a new optional `onNeedName` callback: `() => Promise<string>`.

Execution order:
1. Debounce fires
2. If `versionId` is null (no existing version), call `onNeedName()` and await the returned string
3. Use the returned string as `name` in the `api.saveVersion()` call
4. If `onNeedName` is not provided, fall back to `formData.personalInfo.fullName || 'Untitled CV'` (existing behavior, backward compatible)

The `onNeedName` callback is provided by `DirectEditPage`, which manages the dialog open/close state and resolves a Promise when the user submits or dismisses.

### CVContext ‚Äî No new fields required

`activeVersion.name` is already available. `savedVersions` is already loaded on mount and contains all base CVs for the switcher list.

`handleVersionLoad` in `AppContext` is the existing version-load handler ‚Äî reused as-is for CV switcher item clicks.
