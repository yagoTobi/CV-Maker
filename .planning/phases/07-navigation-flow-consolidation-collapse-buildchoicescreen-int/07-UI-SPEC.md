## Component Inventory

### C-01: LandingScreen (modified)

The landing screen gains two new interaction states: Build Expanded and Tune Expanded. The two-column layout (branding left, actions right) is preserved. Only one card can be expanded at a time.

**Default state (unchanged):** Two stacked cards ("Build my CV", "Tune for a role") plus conditional "My Saved CVs" link.

**Build Expanded state:** When "Build my CV" card is clicked, it expands inline to reveal the import drop zone and "Start from scratch" option directly below it, replacing the separate /build/start route. The "Tune for a role" card collapses to a compact single-line form, or remains visible below.

**Tune Expanded state:** When "Tune for a role" is clicked:
- If exactly 1 base CV exists: navigate directly to `/apply` with that CV's ID. No expansion needed.
- If 0 base CVs exist: expand inline with a message prompting the user to build a CV first, plus a "Build my CV" button.
- If 2+ base CVs exist: expand inline with a lightweight CV picker list.

## Interaction Contracts

### I-01: Build card click

1. User clicks "Build my CV" card on landing.
2. If Build panel is already expanded: collapse it (exit animation).
3. If Tune panel is expanded: collapse Tune panel (exit animation, 200ms), then expand Build panel (enter animation, 250ms).
4. If nothing is expanded: expand Build panel (enter animation, 250ms).
5. Card arrow indicator rotates 90deg clockwise during expansion.
6. `setFormData(null)` is called on expansion (clear any previous form data).

### I-02: Tune card click (0 base CVs)

1. User clicks "Tune for a role" card.
2. `savedVersions` from context is checked. Filter to base CVs only: versions where `parentVersionId` is null/undefined.
3. 0 base CVs found: expand Tune empty-state panel (C-03).
4. Panel shows "No CV to tune yet" with "Build my CV" CTA.

### I-03: Tune card click (exactly 1 base CV)

1. User clicks "Tune for a role" card.
2. 1 base CV found: navigate directly to `/apply` with `state: { baseVersionId: theOnlyBaseCV.id }`.
3. No panel expansion -- direct navigation.

### I-04: Tune card click (2+ base CVs)

1. User clicks "Tune for a role" card.
2. 2+ base CVs found: expand Tune CV picker panel (C-04).
3. Panel shows "Choose a base CV" heading with list of base CVs.
4. User clicks a CV item: navigate to `/apply` with `state: { baseVersionId: selectedCV.id }`.

### I-05: Build panel import flow

1. User drags file onto drop zone or clicks to browse.
2. File validation runs (same as current useFileUpload: .pdf, .docx, .json; max 10MB).
3. On valid file: import begins, drop zone switches to loading state with spinner + progress.
4. On import success: `setFormData(importResult.formData)`, navigate to `/build` (template selector).
5. On import error: error message appears below drop zone.
6. On "Start from scratch" click: `setFormData(null)`, `setSelectedTemplateForBuild(null)`, navigate to `/build`.

### I-06: Panel collapse on navigation

If user navigates away from landing (via "My Saved CVs" link or browser back), any expanded panel should be collapsed. This happens naturally since LandingScreen re-mounts.

### I-07: Route elimination

| Route | Before | After |
|-------|--------|-------|
| `/build/start` | `BuildChoiceScreen` component | Removed. Redirect to `/` or 404 |
| `/` | Landing with `navigate('/build/start')` on Build click | Landing with inline expansion |
| `/dashboard` | Used as intermediate "Tune" step | Management-only, accessible via "My CVs" NavBar link |

## State Requirements

### S-01: Expansion state

LandingScreen maintains local state:

```
expandedPanel: 'build' | 'tune' | null
```

Default: `null`. Toggled on card click. Only one panel can be expanded at a time.

### S-02: Base CV derivation

Base CVs are derived from `savedVersions` by filtering: `savedVersions.filter(v => !v.parentVersionId)`.

This filtering happens on the client using already-loaded context data. No additional API calls needed for the Tune flow decision logic.

### S-03: Import state

Import state (isImporting, importResult, importError, importProgress) comes from the existing `useImport` hook via `useAppContext().cvImport`. The BuildExpansionPanel reuses the same hook instance that BuildChoiceScreen currently uses.

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Card toggle | `aria-expanded="true/false"` on card buttons when they control expansion panels |
| Expansion panel | `role="region"` with `aria-labelledby` pointing to card heading ID |
| CV picker list | `role="list"` on container, `role="listitem"` on each CV row |
| CV picker items | Keyboard-navigable via Tab. Enter/Space activates selection |
| Focus management | When panel expands, focus moves to the first interactive element inside the panel (drop zone for Build, CTA for Tune empty, first CV item for Tune picker) |
| Panel animation | Uses `prefers-reduced-motion: reduce` media query to disable height/opacity animations, showing instant expand/collapse instead |
