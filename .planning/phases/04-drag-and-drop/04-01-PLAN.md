Add `reorderEntries`:
```typescript
const reorderEntries = useCallback(
  (sectionKey: string, from: number, to: number) => {
    setFormData((prev: CVFormData | null) => {
      if (!prev) return prev;
      const next = structuredClone(prev);
      switch (sectionKey) {
        case 'work':
          next.workExperience = reorder(next.workExperience, from, to);
          break;
        case 'education':
          next.education = reorder(next.education, from, to);
          break;
        case 'skills':
          next.skills = reorder(next.skills, from, to);
          break;
        case 'projects':
          next.projects = reorder(next.projects || [], from, to);
          break;
        case 'awards':
          next.awards = reorder(next.awards || [], from, to);
          break;
        default:
          if (sectionKey.startsWith('additional-')) {
            const sIdx = parseInt(sectionKey.split('-')[1], 10);
            const sections = next.additionalSections || [];
            if (sIdx < sections.length) {
              sections[sIdx] = {
                ...sections[sIdx],
                entries: reorder(sections[sIdx].entries, from, to),
              };
              next.additionalSections = sections;
            }
          }
      }
      return next;
    });
  },
  [setFormData]
);
```

Return `reorderSections` and `reorderEntries` from the hook alongside existing returns.

**useSectionDrag.ts** -- Create at `frontend/src/features/direct-edit/hooks/useSectionDrag.ts`. Adapted from the existing `useDrag.ts` pattern in `frontend/src/features/form-builder/components/useDrag.ts`, but scoped to section-level drag using `[data-drag-section]` instead of `[data-drag-card]`.

The hook accepts `onReorder: (from: number, to: number) => void` callback.

State: `dragFromRef = useRef<number | null>(null)`, `dropIndex = useState<number | null>(null)`, `isDragging = useState(false)`.

Returns an object with these handlers:

1. `onGripMouseDown(e: React.MouseEvent)` -- Finds nearest `[data-drag-section]` ancestor via `e.currentTarget.closest('[data-drag-section]')` and sets `element.draggable = true`. Per D-09 contentEditable-safe pattern.

2. `onDragStart(e: React.DragEvent, index: number)` -- Calls `e.stopPropagation()`. Sets `dragFromRef.current = index`. Sets `isDragging(true)`. Suppresses browser ghost: create a 1x1 transparent canvas element via `document.createElement('canvas')`, set width=1, height=1, then call `e.dataTransfer.setDragImage(canvas, 0, 0)`. Set `e.dataTransfer.effectAllowed = 'move'`.

3. `onDragEnter(e: React.DragEvent, index: number)` -- `e.stopPropagation()`. `e.preventDefault()`. If `dragFromRef.current !== null && dragFromRef.current !== index`, set `dropIndex(index)`.

4. `onDragOver(e: React.DragEvent)` -- `e.preventDefault()`. `e.stopPropagation()`.

5. `onDrop(e: React.DragEvent, index: number)` -- `e.stopPropagation()`. If `dragFromRef.current !== null && dragFromRef.current !== index`, call `onReorder(dragFromRef.current, index)`. Reset: `dragFromRef.current = null`, `setDropIndex(null)`, `setIsDragging(false)`.

6. `onDragEnd(e: React.DragEvent)` -- `e.stopPropagation()`. Set `(e.currentTarget as HTMLElement).draggable = false`. Reset: `dragFromRef.current = null`, `setDropIndex(null)`, `setIsDragging(false)`.

7. `dropIndex: number | null` -- Current drop target position for rendering DropLine.

8. `isDragging: boolean` -- Whether a section drag is active (for applying opacity class).

9. `dragFromIndex: number | null` -- Expose `dragFromRef.current` as a computed value (read from ref) so consumers can apply the `.dragging` CSS class to the source item. Implement as a getter or just return `dragFromRef` and let consumer read `.current`.

Actually, since ref.current doesn't trigger re-renders, better to track dragFrom as state too: `const [dragFromIndex, setDragFromIndex] = useState<number | null>(null)`. Update in onDragStart (set to index) and reset in onDrop/onDragEnd (set to null).

Named export: `export function useSectionDrag(onReorder: (from: number, to: number) => void)`.

Return type:
```typescript
{
  onGripMouseDown: (e: React.MouseEvent) => void;
  onDragStart: (e: React.DragEvent, index: number) => void;
  onDragEnter: (e: React.DragEvent, index: number) => void;
  onDragOver: (e: React.DragEvent) => void;
  onDrop: (e: React.DragEvent, index: number) => void;
  onDragEnd: (e: React.DragEvent) => void;
  dropIndex: number | null;
  isDragging: boolean;
  dragFromIndex: number | null;
}
```

**useEntryDrag.ts** -- Create at `frontend/src/features/direct-edit/hooks/useEntryDrag.ts`. Nearly identical to useSectionDrag, but uses `[data-drag-entry]` instead of `[data-drag-section]` for the closest selector in `onGripMouseDown`.

All drag event handlers call `e.stopPropagation()` to prevent bubbling to section-level handlers (per D-06 entry isolation and established useDrag.ts pattern).

The hook accepts `onReorder: (from: number, to: number) => void`.

Same return type as useSectionDrag.

Named export: `export function useEntryDrag(onReorder: (from: number, to: number) => void)`.

**Important implementation notes:**
- Both hooks use `useRef` for the canvas element to avoid creating a new canvas on every drag start. Initialize lazily: `if (!canvasRef.current) { canvasRef.current = document.createElement('canvas'); canvasRef.current.width = 1; canvasRef.current.height = 1; }`.
- Per D-05: The transparent canvas suppresses the browser's default drag ghost image.
- Per D-09: No `draggable` attribute in JSX. Only `onGripMouseDown` sets it dynamically. `onDragEnd` resets it.
