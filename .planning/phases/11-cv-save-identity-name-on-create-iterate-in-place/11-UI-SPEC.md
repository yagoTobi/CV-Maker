# Phase 11 — UI Design Contract

> Visual and interaction contract for the CV Save Identity phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

| Property | Value |

## Spacing Scale

Declared values — multiples of 4, aligned with existing project conventions:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chevron gap in breadcrumb |
| sm | 8px | Prompt input bottom margin, dropdown item vertical padding, button gap |
| md | 16px | Prompt padding horizontal, dropdown padding, NavBar left-group gap (existing) |
| lg | 24px | Prompt padding vertical, dialog max-width gutter |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- NavBar left-group gap: 16px (existing, unchanged — `gap: 16px` in `.leftGroup`)
- NavBar height: 48px (existing, unchanged)
- CV name button padding: 4px 6px (tight fit inside 48px nav row)
- Dropdown item padding: 8px 12px (compact list items)
- Name prompt dialog: 20px 24px padding (slightly wider than ConfirmDialog at 10px 14px, to accommodate input field)

| Body / general UI | 14px | 400 | 1.5 |

| Dropdown item | 13px | 400 | 1.5 |

Source: existing NavBar at 14px/16px; ConfirmDialog at 13px; ghost/accent buttons at 13px.
Two weights used: 400 (regular) and 500 (medium — already loaded in variables.css Google Fonts import).
600 (semibold) reserved for logo only (existing).

## Color

All color values reference existing design token variables. No new hex values introduced.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--bg-primary` | #F8FAFC | Page background |
| Secondary (30%) | `--bg-secondary` | #FFFFFF | NavBar surface, prompt dialog surface, dropdown surface |
| Accent (10%) | `--accent` | #3B82F6 | CV name button on hover/focus, OK button background, breadcrumb separator color |
| Destructive | `--error` | #EF4444 | Not used in this phase (no destructive actions) |

Accent reserved for:
1. CV name button hover text color (matches existing `navLink:hover` pattern)
2. "OK" / "Save" button in name prompt (primary action)
3. Breadcrumb separator "/" character tint (subtle brand touch, see Breadcrumb section)

Secondary semantic note: `--text-secondary` (#64748B) for all non-primary label text — breadcrumb company/role segment, dropdown items at rest, prompt subtext.

Dropdown overlay: no backdrop. The dropdown closes on outside click (click-outside handler) — consistent with the project's preference for minimal overlays (ConfirmDialog has only 0.08 opacity backdrop when it needs one; name prompt uses a semi-transparent backdrop).

## Component Inventory

All new components. No shadcn blocks. No third-party registry.

### 1. NamePromptDialog

**File:** `frontend/src/features/direct-edit/components/NamePromptDialog.tsx` + `NamePromptDialog.module.css`

**Trigger:** Fires once, before the first POST in `useAutoSave` when `activeVersion.id` is null. The prompt intercepts the save, collects a name, then allows the save to proceed.

**Form factor:** Centered minimal dialog (not a full-page modal). Fixed position, centered horizontally and vertically. Semi-transparent backdrop at 0.12 opacity — slightly more visible than ConfirmDialog's 0.08, because this is a purposeful pause rather than a destructive-confirm overlay.

**Anatomy:**
```
┌─────────────────────────────────────┐
│  Give this CV a name                │   ← heading, 14px weight 500, --text-primary
│  You can always rename it later.    │   ← subtext, 13px, --text-secondary
│                                     │
│  ┌───────────────────────────────┐  │
│  │  My CV                        │  │   ← text input, placeholder "My CV"
│  └───────────────────────────────┘  │
│                                     │
│                [Skip]  [Save CV]    │   ← right-aligned buttons
└─────────────────────────────────────┘
```

**Dimensions:** min-width 320px, max-width 420px. Padding 20px 24px. Border-radius `--radius-lg` (12px). Box-shadow `--shadow-lg`. Background `--bg-secondary`.

**Input field:** Full-width, font 14px, padding 8px 10px, border `1px solid --border-color`, border-radius `--radius-sm` (6px). On focus: border-color changes to `--accent`. Placeholder text: "My CV" (or imported filename if available — logic in hook, not visual spec). Value auto-selected on open so user can type immediately without clicking.

**Buttons:**
- "Skip" — ghost style, 13px, `--text-secondary`, matches existing `.cancelBtn` pattern from ConfirmDialog
- "Save CV" — accent style, 13px, `--bg-accent` (#3B82F6) white text, matches existing `.accentBtn` from NavBar
- Button order: Skip left, Save CV right (right-to-left read order for affirmative action)

**Enter key:** Submits the dialog (same as clicking "Save CV").
**Escape key / clicking backdrop:** Dismisses without name — uses "Untitled CV" as fallback (D-04).
**Click on backdrop:** Same as Escape.

**Animation:** `dialogEnter` — identical keyframe to ConfirmDialog: `opacity 0→1, translateY -6px→0, scale 0.97→1, duration 150ms ease-out`. Backdrop fades in at 80ms.

**Accessibility:** `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading. Focus trapped inside dialog while open. Focus returns to editor on close.

### 2. CVNameButton (NavBar left group — editor pages only)

**File:** Inline within `NavBar.tsx` and `NavBar.module.css` (no separate component file — kept simple).

**Condition:** Rendered only when `isEditorPage` is true AND `!isTuning`.

**Replaces:** The "My CVs" link on editor pages (D-05, D-08). On non-editor pages, left group shows logo only.

**Anatomy:**
```
[CV Maker]  [Marketing CV ▾]
  logo       cv name button
```

**Spacing:** `gap: 16px` in `.leftGroup` (existing, unchanged).

**CV name button style:**
- Font: 14px, weight 500, color `--text-primary` at rest
- Padding: 4px 6px
- Background: transparent at rest, `--bg-tertiary` on hover
- Border: none (no border at rest); appears as plain text with subtle hover state
- Border-radius: `--radius-sm` (6px)
- Cursor: pointer
- Chevron: "▾" character (U+25BE, BLACK DOWN-POINTING SMALL TRIANGLE), 10px, `--text-muted`, 4px gap after name
- Transition: `background 150ms ease, color 150ms ease`
- Max-width: 180px; text-overflow: ellipsis; overflow: hidden; white-space: nowrap (truncation for long names)
- On hover: background `--bg-hover`, color `--accent`
- Active/open state (dropdown visible): background `--bg-hover`, color `--accent`, chevron rotates 180deg

**Truncation rule:** Names longer than ~20 characters truncate with ellipsis. Full name shown in `title` attribute for tooltip on hover.

### 3. CVSwitcherDropdown

**File:** `frontend/src/features/direct-edit/components/CVSwitcherDropdown.tsx` + `CVSwitcherDropdown.module.css`

**Trigger:** Click on CVNameButton (toggle). Closes on: outside click, Escape key, selecting an item, CVNameButton click again.

**Positioning:** `position: absolute`, anchored to the left edge of the CVNameButton, `top: calc(100% + 4px)` (4px gap below the NavBar item). Left-aligned with the button. Z-index 100 (above NavBar z-50, below any future modals).

**Container:** Background `--bg-secondary`, border `1px solid --border-color`, border-radius `--radius` (8px), box-shadow `--shadow-lg`, min-width 200px, max-width 280px.

**Animation:** `dropdownEnter` — `opacity 0→1, translateY -6px→0, duration 150ms ease-out`. Matches existing dialog enter patterns.

**List anatomy:**
```
┌───────────────────────────────────────┐
│  Marketing CV           ← active item │   ← current CV, weight 500, accent left border
│  Software Dev CV                      │   ← other base CV
│  General CV                           │   ← other base CV
│  ─────────────────────────────────── │   ← divider
│  + New CV                             │   ← action item
└───────────────────────────────────────┘
```

**Item anatomy:**
- Padding: 8px 12px
- Font: 13px, weight 400, `--text-primary`
- Hover: background `--bg-tertiary`
- Active (current CV being edited): weight 500, left border 2px solid `--accent`, padding-left 10px (2px border + 10px = 12px visual)
- Transition: `background 150ms ease`
- Only base CVs shown (parentVersionId is null) — D-06

**"+ New CV" item:**
- Separated by 1px `--border-color` divider
- Color: `--text-secondary`
- Font: 13px, weight 400
- On click: calls `resetForNewBuild()` and navigates to `/build` (template selector) — consistent with existing "+ New CV" flow from NavBar phase 7

**Empty state (no other base CVs):** Dropdown does not render. CVNameButton remains clickable but shows no dropdown — clicking does nothing (or shows a single-item list with "No other CVs" in `--text-muted` at 13px, non-interactive, then the divider + "+ New CV" action).

**Scroll:** If list exceeds 8 items, max-height 240px, overflow-y auto, custom scrollbar matching `variables.css` scrollbar styles.

**Accessibility:** `role="listbox"` on the list container. Each item has `role="option"`. CVNameButton has `aria-haspopup="listbox"`, `aria-expanded` toggled with open state.

### 4. NavBar Breadcrumb (tuning state)

**File:** Inline within `NavBar.tsx` and `NavBar.module.css`.

**Condition:** Rendered only when `isEditorPage && isTuning` (D-07). Replaces CVNameButton in the left group when tuning.

**Anatomy:**
```
[CV Maker]  [Marketing CV]  /  [Ogilvy]
  logo       base name    sep  company/role
```

**Format:** `[base CV name] / [company or role]`
- Company takes priority over role if both are present
- If neither: show only the base CV name (no separator)

**Base name segment:**
- Font: 14px, weight 500, color `--text-primary`
- Not clickable (static text during tuning — no dropdown while TunePanel is open)
- Max-width: 160px, overflow ellipsis

**Separator "/":**
- Font: 13px, weight 400, color `--accent` (subtle brand, Claude's call per CONTEXT.md)
- Margin: 0 6px (6px either side of the slash)

**Company/role segment:**
- Font: 13px, weight 400, color `--text-secondary`
- Max-width: 160px, overflow ellipsis

**Total breadcrumb max-width:** ~360px (logo 70px + gap 16px + breadcrumb ~360px = fits in 48px nav row without overflow on standard viewports).

**Truncation:** Both segments truncate independently with ellipsis. Neither has a title attribute (tooltips would conflict with TunePanel).

## States and Interactions

### NavBar Left Group — State Machine

| Page / Condition | Left Group Content |
|------------------|--------------------|
| Non-editor page (Landing, Dashboard, TemplateSelector) | Logo only |
| Editor page, not tuning, no activeVersion | Logo + "Untitled CV ▾" (CVNameButton with placeholder name) |
| Editor page, not tuning, activeVersion exists | Logo + "[CV name] ▾" (CVNameButton) |
| Editor page, isTuning | Logo + breadcrumb "[base name] / [company or role]" |

### Name Prompt — State Machine

| Trigger | State |
|---------|-------|
| First auto-save debounce fires, no activeVersion.id | Dialog opens, input auto-focused, text auto-selected |
| User types name and presses Enter or clicks "Save CV" | Dialog closes, name used for version creation, save proceeds |
| User clears input and presses Enter or clicks "Save CV" | Name defaults to "My CV", dialog closes, save proceeds |
| User clicks "Skip" or Escape or backdrop | Dialog closes, name defaults to "Untitled CV", save proceeds |
| Save completes | `activeVersion.id` is set; subsequent saves use PATCH (in-place update per D-02) |

### CV Switcher Dropdown — Interaction Flow

| Action | Result |
|--------|--------|
| Click CVNameButton | Dropdown opens; current CV item is highlighted |
| Click another CV in list | Dropdown closes; `handleVersionLoad(version)` called; editor loads that CV |
| Click "+ New CV" | Dropdown closes; `resetForNewBuild()` + navigate to `/build` |
| Click outside dropdown | Dropdown closes |
| Escape key | Dropdown closes, focus returns to CVNameButton |
| Click CVNameButton again (open) | Dropdown closes |

## Integration Contracts

### EditorActionsContext — Required Extensions

The following fields must be added to the `EditorActions` interface to enable NavBar breadcrumb rendering:

| Field | Type | Source |
|-------|------|--------|
| `cvName` | `string` | `activeVersion?.name ?? 'Untitled CV'` |
| `tuneCompanyName` | `string` | TunePanel tier 2 `companyName` state |
| `tuneRole` | `string` | TunePanel tier 2 `roleName` state |

NavBar reads these fields to render CVNameButton text and the breadcrumb sub-segment.

### useAutoSave — Name Prompt Hook Point

`useAutoSave` receives a new optional `onNeedName` callback: `() => Promise<string>`.

Execution order:
1. Debounce fires
2. If `versionId` is null (no existing version), call `onNeedName()` and await the returned string
3. Use the returned string as `name` in the `api.saveVersion()` call
4. If `onNeedName` is not provided, fall back to `formData.personalInfo.fullName || 'Untitled CV'` (existing behavior, backward compatible)

The `onNeedName` callback is provided by `DirectEditPage`, which manages the dialog open/close state and resolves a Promise when the user submits or dismisses.

### CVContext — No new fields required

`activeVersion.name` is already available. `savedVersions` is already loaded on mount and contains all base CVs for the switcher list.

`handleVersionLoad` in `AppContext` is the existing version-load handler — reused as-is for CV switcher item clicks.
