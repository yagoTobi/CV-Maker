  <action>
    Create MedLengthTemplate.tsx -- Named export `MedLengthTemplate`. This component renders the entire CV document using EditableField for every editable text field.

    **Props interface:**
    ```typescript
    interface MedLengthTemplateProps {
      formData: CVFormData;
      onFieldChange: (path: string, value: string) => void;
      onBulletAdd: (basePath: string, afterIndex: number) => void;
      onBulletRemove: (basePath: string, index: number) => void;
      onInput?: () => void;
    }
    ```

    **Component structure** (mirrors the section_order loop in med-length-proff-cv.tex.j2 lines 39-136):

    1. **Personal Info Header** (always rendered, not in sectionOrder):
       - `<EditableField value={formData.personalInfo.fullName} fieldPath="personalInfo.fullName" tag="h1" className={styles.name} placeholder="Your Name" />`
       - Info bar: Render items based on `formData.personalInfo.personalOrder` (default: phone, email, location, links). For each item, render an EditableField. Between items, render ` | ` separator span. For links, render each link's label as EditableField (NOT as an anchor tag -- per RESEARCH.md Open Question 3: link labels are editable text, URLs are not visible on CV surface).
         - phone: `<EditableField fieldPath="personalInfo.phone" placeholder="+1 (555) 123-4567" />`
         - email: `<EditableField fieldPath="personalInfo.email" placeholder="email@example.com" />`
         - location: `<EditableField fieldPath="personalInfo.location" placeholder="City, State" />`
         - links[i]: `<EditableField fieldPath={`personalInfo.links[${i}].label`} placeholder="Link" className={styles.linkText} />`
       - Summary (if present or always with placeholder): `<EditableField value={formData.personalInfo.summary || ''} fieldPath="personalInfo.summary" tag="p" className={styles.summary} placeholder="Write a brief professional summary..." multiline />`

    2. **Section Loop** -- iterate over `formData.sectionOrder ?? DEFAULT_SECTION_ORDER`:

       **Work section (_sec === 'work'):**
       - Section header: hardcoded "Experience" (matching tex.j2 line 42)
       - For each `job` in `formData.workExperience`:
         - Line 1: Company (bold, EditableField) ... Dates (bold, EditableField startDate + " -- " + EditableField endDate)
         - Line 2: Title (italic, EditableField) ... Location (italic, EditableField)
         - Bullets: `<EditableBulletList bullets={job.bullets} basePath={`workExperience[${i}].bullets`} onBulletChange={(bi, text) => onFieldChange(`workExperience[${i}].bullets[${bi}]`, text)} onBulletAdd={(afterIdx) => onBulletAdd(`workExperience[${i}].bullets`, afterIdx)} onBulletRemove={(bi) => onBulletRemove(`workExperience[${i}].bullets`, bi)} onInput={onInput} />`
         - Use `key={job.id}` on each job div

       **Education section (_sec === 'education'):**
       - Section header: "Education"
       - For each `edu` in `formData.education`:
         - Check if has content items (gpa or details.length > 0). If yes, render full rSubsection layout (same as work). If no, render simplified layout (bold school + dates on line 1, italic degree + location on line 2, with `educationSimple` class).
         - Line 1: School (bold) ... Dates (bold)
         - Line 2: Degree (italic) ... Location (italic)
         - If GPA: render as first bullet "GPA: {gpa}" -- use EditableField for the GPA value
         - Details: `<EditableBulletList bullets={edu.details} basePath={`education[${i}].details`} .../>`
         - Use `key={edu.id}`

       **Skills section (_sec === 'skills'):**
       - Section header: "Skills"
       - Render as grid (`.skillsGrid`). For each `cat` in `formData.skills`:
         - Category label (bold): `<EditableField value={cat.category} fieldPath={`skills[${i}].category`} placeholder="Category" className={styles.skillCategory} />`
         - Skills text: render as a single EditableField with the comma-joined text of `cat.skills.map(s => s.text).join(', ')`. fieldPath = `skills[${i}].skillsText` (a virtual path -- the onFieldChange handler in useDirectEditor will need to handle this by parsing the comma string back into SkillItem[]).

         IMPORTANT: For skills comma text, use a SEPARATE callback or a special field path convention. The simplest approach: use fieldPath `skills[${i}].skillsText` and handle it in onFieldChange at the MedLengthTemplate level. On blur, parse the comma-separated text into SkillItem[] preserving IDs where possible (reuse existing `updateSkillsText` logic from useFormBuilder.ts). Create a local `handleSkillsChange` function for this.

       **Projects section (_sec === 'projects'):**
       - Section header: "Projects"
       - For each `proj` in `formData.projects ?? []`:
         - Project name (bold) ... Year (right-aligned)
         - Description (body text)
         - Technologies (italic, if present)
         - Bullets (if present): `<EditableBulletList .../>`
         - Use `key={proj.id}`

       **Awards section (_sec === 'awards'):**
       - Section header: "Awards"
       - Render as grid (`.awardsGrid`). For each `award` in `formData.awards ?? []`:
         - Year: `<EditableField value={award.year} fieldPath={`awards[${i}].year`} placeholder="Year" />`
         - Title + Description: `<EditableField value={award.title} fieldPath={`awards[${i}].title`} placeholder="Award Title" />` + if description: `: <EditableField value={award.description || ''} fieldPath={`awards[${i}].description`} placeholder="Brief description" />`

       **Additional sections (_sec.startsWith('additional-')):**
       - Parse index from `_sec.split('-')[1]`
       - Section header: EditableField for `additionalSections[idx].title`
       - For each entry, render as rSubsection (same pattern as work)

    **Date rendering pattern** (used in work, education, additional):
    Show start date EditableField, then if either startDate or endDate is non-empty show " -- " separator, then end date EditableField. This matches the LaTeX `\,--\,` separator.

    **Empty section handling:**
    Sections with no entries (e.g., `workExperience.length === 0`) should NOT be rendered. This matches the LaTeX template behavior where `(% if work %)` guards each section.

    **Placeholder text** for each field type per UI-SPEC Copywriting Contract (the exact placeholder strings are specified in the UI-SPEC).
