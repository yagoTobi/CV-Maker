    Implementation per D-07 and D-08:

    - `const timerRef = useRef<ReturnType<typeof setTimeout>>()`

    **useEffect watching formData:**
    ```typescript
    useEffect(() => {
      if (!formData) return;

      const serialized = JSON.stringify(formData);
      if (serialized === lastSavedRef.current) return;
      if (isSavingRef.current) return;

      if (timerRef.current) clearTimeout(timerRef.current);

      timerRef.current = setTimeout(async () => {
        // Pitfall 5: before saving, flush any focused contentEditable element
        const activeEl = document.activeElement;
        if (activeEl?.hasAttribute('data-field-path') && activeEl.textContent != null) {
          // Dispatch a synthetic blur-and-refocus to commit text
          // OR: directly read the text and update formData before saving
          // Implementation choice: trigger a blur event on the active element,
          // which will fire the EditableField's handleBlur -> onFieldChange -> setFormData.
          // Then wait for the next tick before saving.
          // Simpler approach: just read the DOM text. But we need formData to be up-to-date.
          // Best approach per RESEARCH.md recommendation: onInput already updates formData
          // (gated by isFocused for re-render). So formData should be current.
          // If not, blur-and-refocus as fallback.
        }

        isSavingRef.current = true;
        setStatus('saving');
        try {
          const result = await api.saveVersion({
            name: formData.personalInfo.fullName || 'Untitled CV',
            templateId: formData.templateId,
            texContent: '', // LaTeX generated at download time, not during editing
            formData,
            // If versionId exists, we're updating. If not, creating new.
            // Note: api.saveVersion always creates. To update, we'd need PATCH.
            // For now, always POST (creates a new version or updates via id).
          });
          if (result) {
            lastSavedRef.current = serialized;
            setStatus('saved');
          } else {
            setStatus('error');
          }
        } catch {
          setStatus('error');
        } finally {
          isSavingRef.current = false;
        }
      }, 2500); // 2.5 second debounce per D-08

      return () => {
        if (timerRef.current) clearTimeout(timerRef.current);
      };
    }, [formData, versionId]);
    ```

    IMPORTANT NOTE on the save mechanism: The existing `api.saveVersion` always POSTs (creates). For auto-save to work as "update existing version", check if the project has a PATCH endpoint for versions. Looking at the API map: `PATCH /cv-versions/{id}` exists (cv_versions.py:311) but it's for re-parenting, not full updates. So auto-save should:
    - If `versionId` exists: save via whatever update mechanism is available (check if saveVersion with same ID overwrites, or if a separate update function exists)
    - If `versionId` is null: create a new version

    Check `api.saveVersion` implementation -- it POSTs to `/cv-versions`. The backend `POST /cv-versions` creates a new version each time. For auto-save, we want to update the existing version, not create a new one every 2.5 seconds.

    Resolution: The auto-save should save to a draft state. Since the project doesn't have a PATCH for full version updates, the simplest approach for this phase is:
    1. On first save, create a new version and store its ID
    2. On subsequent saves, check if there's an update endpoint. If not, skip the network call and just keep formData in memory (it persists in CVContext).
    3. OR: Add a simple `updateVersion` API method that PATCHes the version.

    The executor should check the actual backend API capabilities and implement accordingly. The hook's contract is: call some save function on debounce, track status. The exact save strategy may need adaptation.

    **SaveIndicator.tsx** -- Named export `SaveIndicator`. Simple presentational component.

    ```typescript
    import type { SaveStatus } from '../hooks/useAutoSave';
    import styles from './SaveIndicator.module.css';

    interface SaveIndicatorProps {
      status: SaveStatus;
    }

    export function SaveIndicator({ status }: SaveIndicatorProps) {
      if (status === 'idle') return null; // Hidden when idle per UI-SPEC Copywriting Contract

      const text = status === 'saving' ? 'Saving...'
                 : status === 'saved'  ? 'Saved'
                 : 'Save failed';

      const className = status === 'saving' ? styles.saving
                      : status === 'saved'  ? styles.saved
                      : styles.error;

      return (
        <div className={`${styles.indicator} ${className}`}>
          {text}
        </div>
      );
    }
    ```
