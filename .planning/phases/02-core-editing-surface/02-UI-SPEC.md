# Phase 2 — UI Design Contract

> Visual and interaction contract for the web-rendered CV editing surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase renders a web CV that looks like the LaTeX PDF output. The CV itself is the interface -- users click on text and type. There are NO form fields, NO side panels, NO toolbars. The only UI chrome is a small save status indicator.

The design contract has two layers:
1. **CV Document Layer** -- EB Garamond, LaTeX-derived spacing, black-on-white. This is the CV itself.
2. **Chrome Layer** -- Save indicator using existing app design tokens (IBM Plex Sans, --accent, etc.).

## Spacing Scale

This phase uses TWO spacing systems. The CV document layer uses LaTeX-derived values (in `pt`, `em`, `in`). The chrome layer uses the existing app spacing tokens.

### CV Document Spacing (derived from resume.cls)

| Token | Value | Usage | Source |
|-------|-------|-------|--------|
| page-pad-y | 0.3in | Top and bottom page padding | resume.cls line 25-26 |
| page-pad-x | 0.2in | Left and right page padding | resume.cls line 27-28 |
| section-list-indent | 0.5em | Left margin inside rSection list | resume.cls line 94 |
| subsection-gap | 0.1em | Space between rSubsection blocks | resume.cls line 112 |
| bullet-indent | 1em | Left margin for bullet lists | resume.cls line 108 |
| bullet-itemsep | -0.4em | Negative margin between bullet items (tight packing) | resume.cls line 109 |
| parskip | 0.54em | Paragraph skip (0.45 * 1.2em baseline) | resume.cls line 31 |

| Token | Value | Usage |
|-------|-------|-------|

| md | 16px | Save indicator margin from edge |

Exceptions: CV document spacing does NOT use the 8-point grid. It uses LaTeX-equivalent values in pt/em/in to achieve ~95% visual fidelity with the PDF output.

## Color

This phase has a unique color profile. The CV document is black text on white paper. The chrome layer uses existing app tokens.

### CV Document Colors

| Role | Value | Usage |
|------|-------|-------|
| Paper (dominant) | `#FFFFFF` | CV background -- white, matching LaTeX output |
| Text | `#000000` | All CV text -- black, matching LaTeX default |
| Link color | `rgb(21, 88, 176)` | Hyperlink text color in personal info bar (matches LaTeX \definecolor{darkblue}) |
| Section rule | `#000000` | Horizontal rule below section headers (0.4pt solid black) |
| Bullet marker | `#000000` | Middle dot bullet character |

### Interaction Colors (for editable field affordances)

| Role | Value | Usage |
|------|-------|-------|
| Focus highlight | `rgba(59, 130, 246, 0.08)` | Background tint on focused editable field (D-01) |
| Hover highlight | `rgba(59, 130, 246, 0.04)` | Background tint on hovered editable field |
| Placeholder text | `#94A3B8` (var(--text-muted)) | CSS ::before placeholder for empty fields (EDIT-04) |

### Chrome Colors (existing app tokens)

| Role | Value | Usage |
|------|-------|-------|
| Save indicator text ("Saved") | `#10B981` (var(--success)) | Saved state |
| Save indicator text ("Saving...") | `#94A3B8` (var(--text-muted)) | Saving state |
| Save indicator text ("Error") | `#EF4444` (var(--error)) | Save error state |

**60/30/10 split for CV document:** 95% white (paper), 4.9% black (text, rules, bullets), 0.1% blue (link color only). This is a document rendering -- the standard 60/30/10 app color split does not apply.

Accent reserved for: Focus highlight on editable fields ONLY. No accent color on the CV document itself.

## Interaction Contract

### Editable Field States

Every text field on the CV has these visual states:

| State | Visual Treatment | Cursor |
|-------|-----------------|--------|
| Default | No border, no background. Field looks like static PDF text. | `default` (arrow) |
| Hover | `rgba(59, 130, 246, 0.04)` background tint. 2px border-radius. | `text` (I-beam) |
| Focused | `rgba(59, 130, 246, 0.08)` background tint. 2px border-radius. No outline. | text cursor (blinking caret inside field) |
| Empty + unfocused | Placeholder text in `#94A3B8` italic via CSS `::before` with `data-placeholder` | `text` |
| Empty + focused | Placeholder hidden (transparent). User types into empty field. | text cursor |

**Transition:** `background-color 0.15s ease` on all states.

**Layout compensation:** Editable fields have `padding: 1px 2px` with `margin: -1px -2px` to prevent the hover/focus background from shifting surrounding text.

**Source:** D-01, D-03, EDIT-04, RESEARCH.md Pattern 4.

### Keyboard Behavior

| Context | Key | Action |
|---------|-----|--------|
| Single-line field (name, company, title, dates) | Enter | Commit text, blur field |
| Single-line field | Escape | Revert to last committed value, blur field |
| Single-line field | Tab | Commit text, blur, move focus to next field (browser default) |
| Bullet field | Enter | Commit current bullet, create new empty bullet below, focus it |
| Bullet field | Backspace (empty bullet) | Delete bullet, focus previous bullet's end |
| Bullet field | Backspace (non-empty) | Normal backspace behavior |
| Bullet field | Escape | Revert to last committed value, blur field |
| Skills field (comma-separated text) | Enter | Commit text, blur field |

**Source:** D-04, EDIT-03, EDIT-06, RESEARCH.md Patterns 1-2.

### Auto-Save Behavior

| Trigger | Timing | Indicator |
|---------|--------|-----------|
| Any field edit (onInput event) | 2.5 second debounce after last input | "Saving..." appears |
| Debounce fires | Immediate | "Saving..." text |
| Save succeeds | Immediate | "Saved" text (stays visible) |
| Save fails | Immediate | "Save failed" text in error color |
| No changes since last save | N/A | "Saved" text (or hidden if never saved) |

**Indicator position:** Fixed to top-right corner of viewport. 16px from right edge, 16px from top edge. Z-index above CV content.

**Indicator style:** No background, no border, no container. Plain text only. Font: 13px IBM Plex Sans, weight 400.

**Source:** D-07, D-08, UX-01, RESEARCH.md Pattern 3.

## CV Document Layout (Full-Bleed)

The CV document fills the viewport width up to a maximum. No page shadow, no gray background, no borders.

| Property | Value | Source |
|----------|-------|--------|
| max-width | 8.5in | US Letter width |
| margin | 0 auto | Centered horizontally |
| padding | 0.3in 0.2in | resume.cls geometry |
| background | white (#FFFFFF) | LaTeX output |
| min-height | 100vh | Content scrolls naturally |
| font-family | "EB Garamond Variable", "EB Garamond", Georgia, serif | med-length-proff-cv.tex.j2 |
| font-size | 11pt | \documentclass[11pt]{resume} |
| line-height | 1.2 | LaTeX default for 11pt |
| color | black (#000000) | LaTeX default |
| text-rendering | optimizeLegibility | Better kerning/ligatures for serif font |
| font-kerning | normal | Match LaTeX kerning behavior |

## CV Section Rendering

### Personal Info Header

| Element | Rendering | CSS |
|---------|-----------|-----|
| Full name | Centered, 20.74pt bold | `font-size: 20.74pt; font-weight: bold; text-align: center;` |
| Info bar (phone, email, location, links) | Centered, items separated by ` \| ` with en-space padding | `text-align: center;` Items joined by ` \| ` (literal pipe with spaces) |
| Summary | Regular body text below header | Standard 11pt body |

### Section Headers (rSection)

| Element | Rendering | CSS |
|---------|-----------|-----|
| Section title | Uppercase bold, 11pt | `text-transform: uppercase; font-weight: bold; font-size: 11pt;` |
| Horizontal rule | 0.4pt solid black below title | `border-bottom: 0.4pt solid black;` |
| Gap below header text | 3pt (LaTeX \smallskip at 11pt) | `margin-bottom: 3pt;` on the header text, rule sits below |
| Section content indent | 0.5em left margin | `padding-left: 0.5em;` |

### Subsections (rSubsection) -- Work, Education, Additional

| Line | Left Content | Right Content | Style |
|------|-------------|---------------|-------|
| Line 1 | Company/School (bold) | Dates (bold) | `display: flex; justify-content: space-between; align-items: baseline;` |
| Line 2 | Title/Degree (italic) | Location (italic) | `display: flex; justify-content: space-between; align-items: baseline; font-style: italic;` |
| Bullets | Middle dot + text | N/A | `padding-left: 1em;` bullet marker via `::before { content: "\00B7"; }` |
| Subsection gap | 0.1em after each subsection | N/A | `margin-bottom: 0.1em;` |
| Bullet spacing | -0.4em between items | N/A | `margin-bottom: -0.4em;` on each bullet item |

## Accessibility Notes

- All contentEditable elements have `role` implicitly set by their semantic HTML tags (h1, h2, span, div).
- Editable fields are keyboard-navigable via Tab (browser default contentEditable behavior).
- Placeholder text uses CSS `::before` pseudo-element (not visible to screen readers). The `data-placeholder` attribute is for CSS only, not aria.
- Focus highlight at 8% opacity on blue (#3B82F6) passes WCAG contrast requirements against white background.
- Save indicator uses color + text (not color alone) to communicate state.
