## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01-01 | Tampering / Information Disclosure | LLM cache key (backend/services/llm_cache.py) | mitigate | Fold `user_clarifications` into the cache_key SHA-256 input (PATTERNS.md §Cache-key update). Without this, two users with identical JD but different clarifications would get the same cached AI response — leaks one user's tailored response shape into another user's session. ASVS V8.1 (data classification at rest). |
| T-13-01-02 | Denial of Service | TailorRequest payload size | accept | FastAPI default 1MB request limit applies. user_clarifications is List[str] of typically <10 chip inputs each <500 chars. Worst case ~5KB. ASVS V13.1.4 — within budget. No additional cap needed; revisit if AI latency budget breaks (Phase 5 AI-06). |
| T-13-01-03 | Information Disclosure (Prompt Injection) | TAILOR_SUGGEST_PROMPT user_message interpolation | mitigate | Clarifications are placed in a labeled `## User-Confirmed Clarifications` block and the prompt explicitly instructs the model to treat them as user-confirmed truth and to NOT fabricate beyond what was said. Adversarial prompt injection ("ignore previous instructions") still possible — model output is downstream-validated by `_resolve_path` and `_sanitize_content`, which already reject path-traversal and dangerous LaTeX. ASVS V14.5.1 — defense in depth via output validation. |
| T-13-01-04 | Tampering | Pydantic validation of List[str] | mitigate | `Optional[List[str]]` annotation rejects non-list values and non-string elements at the FastAPI layer (auto-422). Test 5 of the pytest suite covers this. ASVS V5.1.4. |
| T-13-01-05 | Information Disclosure | Logged user_message contents | accept | Existing `logger.info(...)` in tailor.py already logs prompt fragments. user_clarifications follow the same disposition as the JD itself. No new exposure. Revisit if PII handling tightens. |

**High-severity threats blocking the plan:** None. T-13-01-01 mitigation is mandatory and is implemented inline.
