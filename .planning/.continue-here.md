## Critical Anti-Patterns (do NOT repeat these)

- **`git add -u` after `.planning/` writes** — `.planning/` is gitignored; `-u` will silently miss new planning artifacts. Use `git add -f .planning/quick/...SUMMARY.md` explicitly.
- **`git mv` on empty/gitignored dirs** — fails with "source directory is empty" because untracked. Fall back to plain `mv`.
- **Edit tool batching across many docs** — read state requirements break. Use `sed -i ''` (BSD/macOS) for surgical multi-file path replacements.
- **Attributing pre-existing test failures to refactor work** — always run baseline first when failures appear during hygiene work. 16 template-rendering tests are Phase 13 in-flight, not us.
- **Leaving `.mcp.json` / `.planning/STATE.md` staged in tranche commits** — these are pre-existing dirty. Always `git reset HEAD .mcp.json .planning/STATE.md` before commit.
